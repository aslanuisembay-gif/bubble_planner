# Bubble Planner — Сценарий презентации (пошагово)

## 0) Подготовка перед выступлением (5 минут)
- Открой приложение на `http://localhost:9112`.
- Подготовь 3 демо-задачи:
  - одна обычная активная,
  - одна с repeat,
  - одна с reminder.
- Открой вкладку `Tasks` и `Settings -> Feedback`.
- Подготовь 1 слайд с названием: **Bubble Planner: Smart task flow with cloud sync**.

---

## 1) Вступление (30–40 секунд)
**Текст, который можно говорить:**

"Здравствуйте! Меня зовут Aslan Uisembay. Это мой проект Bubble Planner — приложение для управления задачами с простым интерфейсом, календарными режимами, напоминаниями и облачной синхронизацией через Convex. Цель проекта — сделать планирование быстрым и понятным без сложных экранов."

---

## 2) Проблема и идея (40–60 секунд)
"Я заметил, что многим сложно постоянно вести задачи: интерфейсы перегружены, сложно быстро добавить задачу, а напоминания не всегда удобно контролировать. Поэтому я сделал Bubble Planner с акцентом на 3 вещи:
1. быстрый ввод задач,
2. гибкий просмотр (Day / Week / Month),
3. простая обратная связь и облачное хранение."

---

## 3) Демо: основной пользовательский сценарий (2–3 минуты)

### Шаг 1. Логин / старт
- Покажи экран входа (cloud/offline).
- Скажи: "Приложение поддерживает облачный режим через Convex и офлайн-режим для демо." 

### Шаг 2. Навигация по вкладкам
- Покажи `Talk`, `Bubbles`, `Tasks`.
- Скажи: "Я использую табы, чтобы пользователь быстро переключался между режимами работы."

### Шаг 3. Вкладка Tasks
- Покажи верхние контролы:
  - группа `Day / Week / Month`,
  - группа `Active / Done / All`,
  - `Routines` (switch).
- Скажи: "Пользователь может менять режим календаря и фильтр задач в один клик."

### Шаг 4. Быстрые действия задачи
- Открой `...` у задачи.
- По очереди покажи:
  - Edit,
  - Repeat,
  - Schedule,
  - Reminder,
  - Delete/Share.
- Скажи: "Это ускоряет редактирование без перехода на отдельные экраны."

### Шаг 5. Напоминание
- Покажи баннер напоминания на главном экране.
- Скажи: "Напоминания отображаются в стиле системного banner, чтобы пользователь не пропустил задачу."

### Шаг 6. Feedback
- Открой `Settings -> Feedback`.
- Отправь пример сообщения.
- Скажи: "Все отзывы сохраняются централизованно в одной таблице feedback в Convex."

## 8) Q&A — быстрые ответы
**Вопрос:** Почему Convex?
**Ответ:** Быстрое API + realtime + простая модель функций и таблиц.

**Вопрос:** Что если нет интернета?
**Ответ:** Есть офлайн-путь (demo/local), а при облачном подключении данные синхронизируются.

**Вопрос:** Как собирается feedback?
**Ответ:** Через одну мутацию `feedback:submit` в таблицу `feedback`.
